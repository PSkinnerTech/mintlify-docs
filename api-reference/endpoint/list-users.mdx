---
title: "List users"
api: "GET /api/users"
description: "Get a list of users. Regular users only see themselves, admins see all users."
---

## Description

Returns a paginated list of users. Access is controlled based on user role - regular users can only see their own information, while admins and superadmins can see all users in the system.

## Query Parameters

<ParamField query="page" type="number" required default="1">
  Page number for pagination (minimum: 1)
</ParamField>

<ParamField query="pageSize" type="number" required default="20">
  Number of users per page (minimum: 1, maximum: 100)
</ParamField>

## User Fields

<ParamField name="id" type="string">
  Unique user identifier (UUID format)
</ParamField>

<ParamField name="email" type="string">
  User's email address (maximum 255 characters)
</ParamField>

<ParamField name="name" type="string">
  User's full name (maximum 255 characters)
</ParamField>

<ParamField name="cognitoId" type="string">
  AWS Cognito identifier (maximum 255 characters)
</ParamField>

<ParamField name="role" type="string">
  User role: `user`, `student`, `teacher`, `admin`, or `superadmin`
</ParamField>

<ParamField name="onerosterId" type="string | null">
  OneRoster identifier if associated (maximum 255 characters)
</ParamField>

<ParamField name="primaryOrgId" type="string | null">
  Primary organization identifier (maximum 255 characters)
</ParamField>

<ParamField name="isActive" type="boolean">
  Whether the user account is currently active
</ParamField>

<ParamField name="createdAt" type="string">
  ISO 8601 timestamp when the user was created
</ParamField>

<ParamField name="updatedAt" type="string">
  ISO 8601 timestamp when the user was last updated
</ParamField>

## Access Control

- **Regular Users (`user`, `student`, `teacher`)**: Can only see their own user record
- **Administrators (`admin`, `superadmin`)**: Can see all users in the system
- **Organization Scope**: May be limited to users within the same organization

## Use Cases

- **User Management**: Admin interfaces for managing system users
- **Directory Listing**: Display user directories for administrators
- **Profile Management**: Users viewing their own profile information
- **Organization Management**: Managing users within specific organizations

## Pagination

The response includes pagination metadata to help with large user lists:

- `total`: Total number of users
- `page`: Current page number
- `pageSize`: Number of items per page
- `totalPages`: Total number of pages available
