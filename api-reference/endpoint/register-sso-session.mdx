## Description

Registers a new SSO (Single Sign-On) session after successful authentication. This enables seamless authentication across multiple domains and applications within the TimeBack ecosystem.

## Request Body

<ParamField body="fingerprint" type="string" required>
  Device fingerprint for session identification (minimum 1 character)
</ParamField>

<ParamField body="domain" type="string" required>
  Domain URL requesting the session registration (must be valid URI format)
</ParamField>

## Device Fingerprinting

The `fingerprint` parameter should be a unique identifier for the user's device/browser combination. Common approaches include:

- Browser fingerprinting based on user agent, screen resolution, timezone
- Canvas fingerprinting for unique device identification
- Hardware-based identifiers where available

## Use Cases

- **Cross-Domain SSO**: Enable seamless login across multiple applications
- **Session Management**: Track user sessions across different domains
- **Security**: Associate sessions with specific devices for enhanced security
- **User Experience**: Reduce authentication friction in multi-app environments

## Access Control

- **Authentication Required**: Must have valid JWT token
- **Rate Limiting**: May be subject to rate limiting to prevent abuse
- **Domain Validation**: Domains may need to be pre-approved
